Kurzinfos
Technischer Stand (kurz).
  • Auth & Rollen: Session-Login, BCrypt; Rollen: PATIENT, STAFF, ADMIN
  • BCs aktiv: user, vitals, medication, alerts (Alerts/Events basieren auf vitals)
  • BCs geplant: coredata, assignment, history (derzeit Skeleton)
  • UI/Navigation: Menü-/Header-Struktur ist noch nicht einheitlich (Vereinheitlichung der Templates folgt)
  • Datenbank: MySQL im Betrieb; Tests mit H2 (In-Memory). Schema/Seed erfolgt per eigenem Bootstrap (kein Hibernate ddl-auto). Umstellung auf InnoDB steht noch aus.
  • Build & Tests: JUnit 5 / Spring Boot Test; GitHub Actions: mvn clean test
